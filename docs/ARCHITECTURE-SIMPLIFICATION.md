# 架构简化：从6 Agent并行到单Agent统一模式

## 变更概述

本次更新将 life-kline 项目的 AI 分析从 **6 Agent 并行模式** 简化为 **单 Agent 统一模式**，大幅降低成本和复杂度。

## 变更时间

2025年12月22日

## 主要改动

### 1. 新增文件

- **`server/unifiedAnalyzer.js`** - 单Agent统一分析器
  - 合并6个Agent的功能为1个统一的系统提示词
  - 单次API请求完成所有分析维度
  - 支持多模型降级策略

- **`server/analyzeUnifiedStream.js`** - 统一分析流处理器
  - 处理SSE流式响应
  - 支持缓存机制
  - 保持输出格式兼容

- **`docs/API-UNIFIED-ANALYZER.md`** - API文档
  - 详细的使用说明
  - 迁移指南
  - 最佳实践

### 2. 修改文件

- **`server/index.js`**
  - 新增端点: `POST /api/analyze-unified`
  - 导入统一分析流处理器

### 3. 保留文件（向后兼容）

- `server/parallelAnalyzer.js` - 6 Agent并行分析器（保留）
- `server/analyzeParallelStream.js` - 并行分析流处理器（保留）
- `server/agentPrompts.js` - 6个Agent的提示词（保留）

## API端点变更

### 新增端点

```
POST /api/analyze-unified
```

**用途:** 单Agent统一分析（推荐使用）

**特点:**
- API调用成本降低83%
- 输出格式与并行模式完全兼容
- 支持缓存机制
- 代码简单易维护

### 保留端点（向后兼容）

```
POST /api/analyze-parallel
```

**用途:** 6 Agent并行分析（保留兼容）

**特点:**
- 渐进式推送，首个Agent响应更快
- 单个Agent失败不影响整体
- 成本高6倍

## 迁移建议

### 前端代码迁移

**原代码:**
```javascript
// 使用并行模式
const response = await fetch('/api/analyze-parallel', {
  method: 'POST',
  body: JSON.stringify(baziData)
});
```

**新代码:**
```javascript
// 改为统一模式，只需修改端点
const response = await fetch('/api/analyze-unified', {
  method: 'POST',
  body: JSON.stringify(baziData)
});
```

**重要:** 输出格式完全兼容，无需修改后续数据处理逻辑！

### 迁移步骤

1. **测试环境验证**
   - 在测试环境调用 `/api/analyze-unified`
   - 对比输出格式与 `/api/analyze-parallel` 是否一致
   - 验证缓存机制工作正常

2. **灰度发布**
   - 10%流量切换到统一模式
   - 监控错误率和响应时间
   - 对比成本节省

3. **全量切换**
   - 所有新请求使用统一模式
   - 保留并行模式作为降级备份

## 缓存机制

两种模式共享同一套缓存系统：

- **缓存键:** 基于八字四柱的 SHA256 哈希
- **缓存时效:** 永久（八字分析结果不变）
- **缓存命中:** 秒级返回，无API调用成本
- **性别区分:** 男女八字分别缓存

## 降级策略

### 统一模式降级链

1. 主模型: `gemini-3-pro-preview`
2. 备用模型: `grok-4-auto`
3. 备用模型: `claude-haiku-4-5-20251001`
4. 备用模型: `grok-4`
5. K线降级: 本地算法生成

### 失败处理

- 单次请求失败 → 自动重试（最多2次）
- 主模型失败 → 自动切换备用模型
- 所有AI失败 → K线使用本地算法，其他字段返回降级内容

## 监控指标

建议监控以下指标：

1. **成功率**
   - 统一模式成功率目标: >95%
   - 缓存命中率目标: >60%

2. **响应时间**
   - 缓存命中: <1秒
   - 首次分析: 15-20秒
   - P95响应时间: <25秒

3. **成本**
   - API调用次数/天
   - 估算成本/天
   - 节省比例

4. **错误**
   - 超时错误率
   - JSON解析错误率
   - 模型降级次数

## 回滚方案

如统一模式出现严重问题，可立即回滚：

1. 前端切换回 `/api/analyze-parallel`
2. 无需重启服务器
3. 无需修改数据库
4. 缓存数据继续有效

## 技术细节

### 统一提示词设计

合并了6个Agent的功能要求：

```
1. 核心命理分析 (原Agent1)
   - 日主强弱、十神配置、用神喜忌
   - 性格深度剖析、六亲关系
   - 风水建议、个人特征

2. 100年K线数据 (原Agent2A + 2B)
   - 流年干支分析
   - 运势评分计算 (0-100)
   - K线数据生成 (OHLC)
   - 关键年份标记

3. 事业财富分析 (原Agent3)
   - 官杀星、印星、财星分析
   - 具体行业推荐
   - 财富层级判断

4. 婚姻健康分析 (原Agent4)
   - 配偶星、婚姻宫分析
   - 健康五脏对应
   - 婚姻时机预测

5. 币圈交易分析 (原Agent5)
   - 偏财星、七杀分析
   - 交易风格判断
   - 暴富流年预测

6. 运势预测
   - 月运、年运分析
   - 幸运元素推荐
```

### 数据验证

返回数据必须包含：
- `summary`, `personality`, `industry`, `wealth`
- `marriage`, `health`, `crypto`
- `chartPoints` (至少50年数据)

否则触发降级或重试。

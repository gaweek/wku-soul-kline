# WKU soul-kline｜谁会再次想起你？

## 1. 作品开场

谁会停下来看你，谁会再次想起你，谁又真正懂你的灵魂？

你不是不想社交，而是还没遇到那个真正懂你灵魂的人。

**WKU soul-kline 是一个面向 Soul 兴趣社交场景的 AI 连接分析产品。它把生日、性别、MBTI、SBTI、兴趣和真实社交样本，转译成一条可以触摸的社交连接 K 线。**

它不判断你是谁，也不做命运预测。它既能回答“谁更懂我”，也能回答“我们怎么靠近”：一个陌生人如何从第一眼看见你，到愿意靠近你，再到再次想起你。

### 2.1 产品定位

WKU soul-kline 解决的是兴趣社交里的一个核心问题：用户想被理解，也想遇见同频的人，但常常不知道自己如何被看见、如何开场、如何让一段连接继续发生。

产品把“社交连接”拆成可读的过程：

| 阶段 | 用户关心的问题 |
| --- | --- |
| 第一眼 | 什么会让别人停下来看我？ |
| 接住 | 哪些兴趣和表达更容易被理解？ |
| 共振 | 我和对方在哪里同频？ |
| 错频 | 哪些话题或状态容易被误读？ |
| 再次想起 | 什么会成为对方记住我的理由？ |

### 2.2 创意闪念

谁会停下来看你，谁会再次想起你，谁又真正懂你的灵魂？WKU 把生日、性别、MBTI、SBTI、兴趣和真实社交样本，画成一条可触摸的 soul-kline。它不判断你是谁，只复原陌生人从第一眼、愿意靠近到再次想起你的连接过程。

## 3. 用户故事

一个 Soul 用户想认识更同频的人，但又不想把自己包装成标准答案。

他输入自己的生日、性别、MBTI、SBTI、兴趣、当前社交状态，以及几段真实互动样本。WKU 先生成个人 soul-kline，告诉他哪些兴趣更容易让别人停下来看，哪些表达容易被接住，哪些地方可能造成错频。

当他遇到一个想继续了解的人，可以进入 `Who Know Us`。双方输入各自样本后，WKU 不会给出“配不配”的结论，而是生成一条双人共振 K 线：从眼缘停留、想再看看、第一句话、同频点亮、慢慢深聊，到再次想起。

这条线会告诉两个人：共同点在哪里，哪一刻容易错频，第一句话可以怎么说，以及这段关系适合用什么节奏靠近。用户不再只是被动等待对方开口，而是拿到一个轻松、具体、不冒犯的连接入口。

## 4. 核心功能

### 4.1 Who Know U：单人连接分析

`Who Know U` 用来回答“谁更懂我”。

输入内容：

- 生日、性别。
- MBTI、SBTI。
- 兴趣标签。
- 当前社交状态。
- 真实社交样本。

输出内容：

- 个人 soul-kline。
- 被看见的高点和被误读的低点。
- 更容易产生同频的人群线索。
- 社交表达建议。
- 安全边界提醒。

### 4.2 Who Know Us：双人共振分析

`Who Know Us` 用来回答“我们怎么靠近”。

输入两个人的社交样本后，系统会生成双人共振 K 线。它的重点不是判断关系结果，而是把两个人从“互相注意”到“再次想起”的靠近路径拆出来。

输入内容：

- A/B 双方的生日、性别、MBTI、SBTI。
- A/B 双方的兴趣标签和当前社交状态。
- A/B 双方的真实社交样本。
- 两个人希望了解或靠近的关系目标。

输出内容：

- 双人共振 K 线。
- A/B 各自的连接画像摘要。
- 共振分和双人共振类型。
- 共同点与同频入口。
- 错频风险和降低误读的方法。
- 可直接使用的聊天桥接话题。
- 六个阶段的靠近建议。

这个模式不输出“匹配 / 不匹配”的审判，而是给出更适合兴趣社交的连接路径。

### 4.3 双人共振 K 线

双人共振 K 线包含 6 个主阶段，每个阶段有 3 个微节点，共 18 个连续节点。

| 阶段 | 关注的问题 | 输出示例 |
| --- | --- | --- |
| 眼缘停留 | 两个人是否愿意多看一眼 | 第一眼亮点、停留原因 |
| 想再看看 | 对方身上是否有继续了解的理由 | 兴趣钩子、记忆点 |
| 第一句话 | 怎么开口不尴尬 | 轻松第一句话、低压开场 |
| 同频点亮 | 哪些内容会让两个人更快靠近 | 共同兴趣、表达节奏 |
| 慢慢深聊 | 如何从浅聊进入更真实的交流 | 适合展开的话题、边界提醒 |
| 再次想起 | 什么会成为下一次互动的理由 | 记忆点、再次联系建议 |

K 线纵轴不是“关系预测”，而是该节点继续靠近的共振势能。它让用户看到：这段连接在哪里升温，哪里容易回落，哪里值得再靠近一次。

### 4.4 聊天桥接与错频提醒

`Who Know Us` 会把双人共振落到可执行的聊天建议上。

| 输出 | 作用 |
| --- | --- |
| overlapSignals | 找出两个人一起变熟的入口 |
| mismatchRisks | 标出容易误会或错频的瞬间 |
| conversationBridges | 生成可以直接使用的第一句话 |
| stageAdvice | 按六个阶段给出不制造压力的靠近建议 |
| safety | 检查隐私暴露、操控性话术和过度关系承诺 |

例如，它不会说“你们很适合”，而会说“可以从共同兴趣里的某个轻话题开始，先用低压方式确认对方是否愿意继续聊”。

### 4.5 可交互 soul-kline

生成结果不是静态报告，而是一条可以交互查看的曲线。

用户可以：

- hover 节点查看解释。
- 点击节点锁定详情。
- 看到每个连接阶段的变化。
- 从曲线高低理解一段关系的升温、回落和再次靠近机会。

### 4.6 多 Agent 流式生成

后端通过多 Agent 协作生成分析结果，前端用 SSE 展示生成进度。

用户看到的不是“等待 AI 返回答案”，而是一个逐步发生的分析过程。

## 5. 核心流程

### 5.1 单人分析流程

| 步骤 | 用户动作 | 系统反馈 |
| --- | --- | --- |
| 1 | 选择 `Who Know U` | 进入单人连接分析 |
| 2 | 填写生日、性别、MBTI、SBTI、兴趣和社交样本 | 系统清洗输入并启动 Agent 分析 |
| 3 | 等待生成 | 前端流式展示 Agent 进度和预览结果 |
| 4 | 查看 soul-kline | 曲线展示被看见、接住、错频、再次想起等节点 |
| 5 | 点击或 hover 节点 | 查看节点解释、风险和建议 |

### 5.2 双人共振流程

| 步骤 | 用户动作 | 系统反馈 |
| --- | --- | --- |
| 1 | 选择 `Who Know Us` | 进入双人共振分析 |
| 2 | 输入 A/B 双方信息和真实社交样本 | 系统分别建模两个人的连接画像 |
| 3 | 启动双人 Agent 分析 | 分析共同兴趣、表达节奏、关系目标和错频风险 |
| 4 | 查看双人共振 K 线 | 展示 6 个阶段、18 个节点的靠近路径 |
| 5 | 查看共振解读 | 输出共振分、双人共振类型、共同点和错频点 |
| 6 | 使用聊天桥接话题 | 获得可直接开口的第一句话和下一步靠近建议 |

## 6. 产品亮点

### 6.1 把抽象连接变成可视化体验

“懂不懂我”通常很抽象。WKU soul-kline 把它变成一条可读、可点、可分享的曲线，让用户看见连接是如何发生的。

### 6.2 从匹配判断转向靠近建议

产品不做简单的匹配分数，也不告诉用户“你们合不合适”。它给出的是共同点、错频风险和下一句可以怎么聊。

### 6.3 单人分析自然引出双人邀请

用户先看自己的 soul-kline，再邀请另一个人一起看 `Who Know Us`。这让 AI 分析从单次报告变成社交邀请。

### 6.4 双人共振不制造关系压力

`Who Know Us` 不给关系盖章，也不制造“命中注定”的暗示。它只把两个人目前可见的共同点、错频风险和靠近节奏讲清楚，让用户有一个自然开口的理由。

### 6.5 更适合 Soul 的情绪入口

“谁会再次想起你”比“人格测试”更贴近兴趣社交中的真实动机：用户想被看见，也想遇见真正懂自己的人。

### 6.6 有边界的 AI 社交分析

生日、MBTI、SBTI、兴趣都只是表达语境，不是决定性标签。系统避免命运预测、人格定型和关系审判。

## 7. AI 分析结构

### 7.1 单人连接分析 Agent

| Agent | 作用 |
| --- | --- |
| 画像建模 Agent | 提取兴趣、表达习惯和社交状态。 |
| 共鸣信号 Agent | 判断哪些信息更容易触发同频。 |
| soul-kline Agent | 把连接过程转成曲线节点。 |
| 同频人群 Agent | 推断更可能停下来看用户的人群线索。 |
| 表达方案 Agent | 生成破冰话题和靠近建议。 |
| 安全边界 Agent | 避免命运预测、人格定型和过度判断。 |

### 7.2 双人共振 Agent

| 双人 Agent | 作用 |
| --- | --- |
| Pair Asset Agent | 分别提取 A/B 的连接类型、关键词和靠近方式 |
| Resonance Agent | 分析同频入口、错频风险和第一句话 |
| Resonance K-Line Agent | 生成 18 个连续节点的双人共振 K 线 |
| Context Agent | 输出共振分、共振类型和整体关系语境 |
| Stage Advice Agent | 按六个阶段生成靠近建议 |
| Safety Agent | 检查隐私、操控性话术和过度关系承诺 |

## 11. 安全边界

- 不做命运预测。
- 不做医疗判断或心理诊断。
- 不把 MBTI、SBTI、星座作为决定性结论。
- 不输出“匹配 / 不匹配”的关系审判。
- 所有结论都回到社交表达、共同点、错频风险和聊天建议。

# UI 重构方案（参考 x.com 三栏布局）

## 1. 背景与目标
当前项目的功能链路已经非常完整：
- 生成：`/` 通过 `POST /api/analyze-stream` 进行 SSE 流式生成命理报告（图表 + 多维度分析卡片）
- 导出：打印 PDF、保存 HTML、导出图片
- 传播：Twitter/X、Telegram、微信二维码、复制链接
- 内容沉淀：知识中心（`/knowledge`）与案例库（`/cases`）

问题主要集中在「信息架构与界面合理性」：页面更像“落地页 + 工具”，而不是一个可以持续复访、持续消费内容、持续分享传播的产品。

### 核心目标
- **目标 A（增长）**：让“社媒传播 → 打开链接 → 生成报告 → 一键分享回社媒”的路径更短、更像 x.com 的使用体验
- **目标 B（留存）**：让“生成结果”成为可复访的资产（我的报告、历史、收藏/置顶）
- **目标 C（转化）**：把游客模式的权益边界通过右侧栏与按钮锁定策略表达清楚（减少打断、提升转化）

## 2. 关键用户路径（User Journey）
### 2.1 社媒引流（主路径）
- 用户从 x.com / Telegram 点击链接进入
- 首屏在主栏顶部看到 **Composer（生成入口）**，无需理解复杂信息架构
- 生成完成后得到 **ReportCard（可分享卡片）** 与 **Report Viewer（详情）**
- 一键分享到 X / TG（或导出图片后发帖）

### 2.2 复访留存（次路径）
- 再次进入后默认看到 **Mine（我的）** Tab：最近报告、历史、可再次打开
- 从右侧栏/推荐卡片进入知识/案例，继续消费内容

### 2.3 权益转化（付费/登录）
- 游客可生成（或有限生成）
- 当点击“保存 HTML / 打印 PDF / 跨设备历史”等能力时：
  - **按钮锁定 + 解释 tooltip**
  - 点击后再弹登录（减少频繁弹窗）

## 3. 信息架构与全局布局（参考 x.com）
### 3.1 桌面端：三栏固定布局
- **左侧栏（Left Nav）**：固定/粘性，承载全局导航 + 用户状态
- **中间主栏（Main）**：信息流 + 生成入口 + 详情阅读
- **右侧栏（Right Sidebar）**：趋势/推荐/CTA/社群入口/权益说明（增长与转化核心）

布局示意：

```text
+-------------------+--------------------------+--------------------+
| Left Nav (sticky) | Main Column (scroll)     | Right Sidebar      |
| - Logo            | - Tabs: For You / Mine   | - CTA: 立即生成     |
| - Generate        | - Composer (生成入口)     | - 登录/点数卡片      |
| - My Reports      | - Feed List (卡片信息流)  | - Trending         |
| - Knowledge       | - Detail Drawer/Route    | - 推荐：知识/案例    |
| - Cases           |                          | - TG / X links     |
+-------------------+--------------------------+--------------------+
```

### 3.2 移动端：底部导航 + 单栏
- 采用 x.com 类似的底部 Tab Bar
- 右侧栏的内容（Trending/CTA）在移动端以“插入信息流的卡片”形式出现，或单独做“探索”页

建议底部 Tab：
- 生成
- 我的
- 知识
- 案例
- 账号

## 4. 路由规划（兼容现有路由，逐步迁移）
当前已存在：
- `/` 生成与结果
- `/knowledge`、`/knowledge/:slug`
- `/cases`、`/cases/:id`

建议新增（可逐步上）：
- `/feed`（可选，或者让 `/` 直接变为 Feed）
- `/generate`（可选：沉浸式生成 Studio）
- `/reports/:id`（中长期：报告详情页，便于分享与 SEO）
- `/search`（聚合搜索：知识/案例/未来报告）

> 迁移策略建议：优先改“布局 + 主栏信息流 + Composer”，先不动已有内容页结构。

## 5. 主栏（Main Column）设计
### 5.1 Tabs（For You / Mine）
- **For You**：推荐内容（案例精选/知识精选/公告）
- **Mine**：我的报告（本地历史 + 登录后服务器历史）

### 5.2 Composer（生成入口）
把当前 `/` 的“Hero + 表单”重塑成 x.com 的“发帖框”形态：
- 默认折叠（仅显示最关键输入）
- 支持展开：高级选项（自定义 API、模型等）
- 提交后展示 SSE 进度（来自 `analyze-stream` 的 progress event）
- 完成后生成一个 ReportCard 并置顶（或自动打开详情）

### 5.3 Feed 卡片类型（统一卡片体系）
建议在信息流中统一使用 Card 组件承载不同内容：
- **ReportCard**：报告摘要卡片（含快速动作：分享/导出/查看）
- **CaseCard**：案例卡片（现有案例库卡片风格可复用）
- **KnowledgeCard**：知识文章卡片
- **AnnouncementCard**：公告/增长卡片（社群、关注、权益）

### 5.4 Report Viewer（报告详情）
推荐交互：
- 点击 ReportCard 打开 **Drawer / Modal**（贴近 x.com 详情打开体验）
- 或跳转 `/reports/:id`

报告详情内应提供固定 Action Bar：
- 分享（X/TG/二维码/复制链接/导出图片）
- 保存 HTML
- 打印 PDF
- 重新生成（带入上次输入）

## 6. 右侧栏（Right Sidebar）设计（增长与转化核心）
右侧栏建议固定放置以下模块：
- **CTA Card：立即生成**（对未生成用户尤其重要）
- **账号/点数卡片**：
  - 已登录：显示点数、最近报告入口
  - 游客：展示“登录可获得：保存/打印/跨设备历史”等权益
- **Trending**：热门案例类型、热门知识分类
- **社群入口**：Telegram、x.com
- **轻量免责声明**：避免占用过多空间

## 7. UI 风格与视觉要点（x.com 取向）
- 更“信息密度”与“可扫读”
- 标题、摘要、元信息（时间/浏览量/评分）呈现层级清晰
- 动作按钮保持统一（Primary / Secondary / Tertiary）
- 对“锁定能力”使用一致的视觉语言（锁图标 + tooltip + 点击触发登录）

## 8. 风险与注意事项
- **分享组件一致性**：当前代码中 `SharePanel` 的 props 形态在不同使用处存在不一致风险（建议在落地阶段统一一个 Share API 与组件接口）。
- **报告可复访性**：目前“报告”主要依赖 localStorage history 与当前页面状态。要完整实现 `/reports/:id` 与可分享 URL，需要后端提供 report 存储与权限/分享 token。

## 9. 交付物定义（验收标准）
- 布局变为三栏（桌面）+ 底部导航（移动）
- `/` 首屏为：Tabs + Composer + Feed
- 生成后在 Feed 中出现 ReportCard，且可一键分享与打开详情
- 右侧栏包含 CTA/账号/推荐/社群模块
